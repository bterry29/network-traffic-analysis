# Linux Commands Used

- sudo apt install wireshark
- sudo apt install tcpdump
- sudo apt install dumpcap
- ip addr show
- ifconfig (network information of macOS)
- ip route
- ping -c 4 8.8.8.8 (sends 4 ICMP packets to confirm active machine network traffic capture)
- sudo apt install -y suricata
- sudo suricata
- sudo suricata-update (updates Suricata's ruleset to analyze up-to-date threats) 
- sudo suricata -c /etc/suricata/suricata.yaml -i enp0s1 (runs Suricata and allows it to analyze network traffic information from the interface enp0s1)
- cat /var/log/suricata/fast.log (reads the alerts from previous network traffic analyzed) 
- sudo adduser (create new user for testing ssh attacks) 
- sudo passwd <usermame> (sets password for created user) 
- sudo apt install -y openssh-server
- sudo apt install -y apache2
- sudo apt install curl

#Attack Commands Used (from ATTACK MACHINE to TARGET MACHINE) 

- ping -c 4 <target_IP> (not an attack but shows connection between machines)
- nmap -sn <target_IP> (host scan; MAC address request)
- nmmap -St <target_IP> (full TCP handshake, all 1000 ports)
- nmap -sS -p 22,80 <target_IP> (stealth SYN scan on ports 22 and 80, SSH and HTTP respecitvely) 
- nmap -sS -Pn -0 22,80 --min-rate 50 --max-retries 1 (target_IP> (skip host discovery, speeds up packet rate and keeps it short and controlled)
- hyrda -l testuser -P ~/small_wordlist.txt -t 1 -f -V <target_IP> ssh (uses the password list small_wordlist.txt to attempt to gain access to user "testuser")
